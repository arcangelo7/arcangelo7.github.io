\section{Summary}

The third year of the PhD program in Cultural Heritage in the Digital Ecosystem focused on developing production systems for scholarly data management. Two platforms were advanced: HERITRACE, a web platform for scholarly data curation deployed at the University of Bologna, and OpenCitations Meta, a bibliographic metadata index. Research continued on temporal data management, semantic data extraction methodologies, and supporting infrastructure development.

\section{Main activities}

\subsection{HERITRACE platform development}

HERITRACE was deployed at \url{https://paratext.ficlit.unibo.it/} in October 2024 for the ParaText project, a PRIN 2022 research initiative investigating relationships between literary texts and exegetical paratexts in ancient Greek poetry manuscript transmission. The platform serves as a semantic data editor for the ParaText Bibliographical Database, enabling Classical Philology scholars to manage complex bibliographic relationships with provenance and change-tracking.

The architecture uses Docker-based deployment supporting Virtuoso and Blazegraph triplestores. A Redis-based resource locking mechanism was implemented to handle concurrent modifications. Authentication is provided through ORCID integration, linking modifications to researchers' scholarly identities.

The platform supports entity creation both with and without predefined schemas, accommodating structured data with SHACL validation and unstructured data with dynamic type selection. Display rules are configurable per class or shape for visualization flexibility.

The search functionality switches between SPARQL regex queries and Virtuoso's text index depending on query characteristics. The system handles orphaned entities and cascade deletions while maintaining referential integrity. Usability testing conducted in August/September 2025 informed interface modifications.

A documentation system based on Starlight/MDX was developed to provide technical documentation for HERITRACE, supporting community adoption and maintenance of the platform.

\subsection{OpenCitations Meta}

The OpenCitations Meta index underwent performance optimizations. The system now operates exclusively through triplestores for both data and provenance, addressing previous file-based limitations.

A merging system was developed to address existing duplicates in the index. The software identifies and merges duplicate entities based exclusively on shared identifiers.

The infrastructure includes workflow documentation, automated testing, and semantic versioning with automated releases to support maintainability of the data processing pipeline.

A suite of Virtuoso utilities was developed including Docker-based deployment automation, bulk loading tools, and database optimization utilities to support large-scale triple/quadstore operations in the OpenCitations infrastructure.

\subsection{Temporal data management research}

The integration of the Time Agnostic Library within HERITRACE revealed performance bottlenecks related to deleted entity identification. The HERITRACE Time Vault feature, which displays deleted entities, requires reconstructing historical snapshots for each deleted entity to determine its class information, creating performance barriers at scale.

Evaluation of the Time Agnostic Library was constrained by limited benchmark availability. Only three benchmarks exist for temporal RDF systems: EvoGen, BEAR, and SPBv. EvoGen lacks community adoption preventing cross-system comparisons. BEAR's datasets are no longer accessible online despite attempts to contact authors. SPBv requires implementing custom Java components to interface with the HOBBIT benchmark framework.

Development efforts to create the required Java component for HOBBIT integration proved complex due to platform-specific requirements for component interaction and data serialization. The HOBBIT platform itself exhibits stability issues: previously published benchmarks no longer execute successfully on the current platform, indicating systemic problems that prevented meaningful performance evaluation against other temporal RDF systems.

\subsection{RML and knowledge graph inversion}

Research on knowledge graph inversion was conducted as part of the joint PhD program with KU Leuven under the supervision of Anastasia Dimou. This work addresses the bidirectional transformation challenge: while RML (RDF Mapping Language) enables conversion from relational databases to RDF, the inverse process -- converting RDF back to relational structures using the same mapping definitions -- remained unexplored.

The developed system reuses existing RML mappings to perform reverse transformations, maintaining consistency between forward and backward conversions. Comprehensive testing verified coverage of most R2RML specification use cases for relational databases. Performance evaluation employed the KROWN benchmark, originally designed for RML processors like Morph-KGC. This benchmark adaptation quantifies both the system's scalability characteristics and the computational overhead of bidirectional transformation compared to unidirectional RML processing.

\subsection{Supporting infrastructure and tools}

The SKG-IF SHACL extractor was developed to automatically generate SHACL shapes from annotated ontologies. The system parses ontology constraints and transforms them into corresponding validation shapes, eliminating manual shape creation.

A crowdsourcing system was developed in partnership with PKP (Public Knowledge Project) and TIB (Technische Informationsbibliothek) to integrate citation submission workflows into Open Journal Systems (OJS). The system addresses the visibility gap affecting 52,000 active OJS journals worldwide, with 79.9\% located in the Global South but largely absent from major citation indexes.

The workflow enables journal editors to contribute citations by copying references from articles into OJS dashboard fields. The system parses references to extract structured metadata, allowing manual error correction before submission. Citations are formatted as CSV files and submitted to GitHub as structured issues. Validation checks CSV structure, validates identifiers, and ensures metadata completeness.
